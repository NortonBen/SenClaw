# Channel App Design Specification

## 1. Introduction
The Channel App is a secure, real-time client for SenClaw. It enables users to interact with their AI agents on the go without compromising privacy.

## 2. Network Architecture
The connection follows a relay model to handle NAT traversal and provide a stable public endpoint for mobile devices.

### 2.1. Components
- **SenClaw (Edge)**: The instance where agents run. Connects as a gRPC client to ClawHub.
- **ClawHub (Relay)**: A public server that routes gRPC streams between Edge instances and Mobile Apps.
- **Channel App (Client)**: Flutter app for Android and macOS.

### 2.2. gRPC Stream Definition
The bi-directional stream allows for:
- Real-time message delivery.
- Typing indicators.
- Status updates (agent state).
- Permission requests.

## 3. Security & Privacy
### 3.1. End-to-End Encryption (E2EE)
- **Shared Secret**: A 256-bit key generated by SenClaw and shared via QR.
- **Protocol**: AES-256-GCM.
- **Encrypted Content**:
    - Message body (text/markdown).
    - Tool call details.
    - File metadata.
- **Unencrypted Metadata (for routing)**:
    - `channel_id`.
    - `message_id`.
    - `timestamp`.
    - `sender_type` (Agent/User).

### 3.2. Authentication
- **Edge Auth**: SenClaw uses its `clh_...` token to authenticate with ClawHub.
- **App Auth**: App uses a scoped token derived from the QR pairing process.

## 4. Onboarding Workflow
1. User goes to SenClaw Web UI -> "Connect Mobile App".
2. SenClaw generates a unique `channel_id` and a random `encryption_key`.
3. Web UI displays a QR code.
4. App scans QR -> validates signature -> stores credentials in secure storage (Keychain/Keystore).
5. App initiates gRPC connection to the `hub` URL.

## 5. App Features (Flutter)
- **Multi-Agent Support**: Switch between different agents/groups.
- **Rich Messaging**: Markdown support, code highlighting.
- **History Sync**: Fetch last N messages on startup via OpenAPI, then listen via gRPC.
- **System Commands**: Restart agent, clear memory, etc.
- **Push Notifications**: (Future) Relayed through ClawHub using FCM/APNs.

## 6. Communication Protocol (Protobuf)
```protobuf
syntax = "proto3";

package senclaw.relay;

service ChannelRelay {
    // Bi-directional stream for real-time interaction
    rpc Stream(stream RelayMessage) returns (stream RelayMessage);
}

message RelayMessage {
    string channel_id = 1;
    string sender_id = 2;
    int64 timestamp = 3;
    
    oneof payload {
        EncryptedData encrypted_data = 4;
        ControlMessage control = 5;
    }
}

message EncryptedData {
    bytes nonce = 1;
    bytes ciphertext = 2;
    bytes tag = 3;
}

message ControlMessage {
    enum Type {
        PING = 0;
        PONG = 1;
        ACK = 2;
        TYPING_START = 3;
        TYPING_STOP = 4;
    }
    Type type = 1;
}
```
