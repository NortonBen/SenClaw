# Telegram Channel Binding Guide

## Prerequisites

1. In Telegram, open [@BotFather](https://t.me/BotFather) and send `/newbot` to create a bot, then copy the **Bot Token**.
2. Get your **Telegram User ID** (for example, by messaging [@userinfobot](https://t.me/userinfobot)).

---

## Method 1: Primary Bot (`.env` configuration)

Use this for the first / primary bot binding. It binds to `agents/main/` by default.

```bash
# .env
TELEGRAM_BOT_TOKEN=123456:ABC-your-token
ADMIN_TELEGRAM_USER_ID=987654321

# Optional: bind to a different folder (default: main)
# TELEGRAM_AGENT_FOLDER=main
```

Restart `senclaw` after editing to apply changes.

---

## Method 2: Additional Bots (Web UI or CLI)

Use this to bind second/third bots to different agent folders. Config is saved to `~/.senclaw/config.json` and takes effect immediately without restart.

### Web UI Configuration

Open Settings -> Agents -> **Add Agent**, then set Channel to `Telegram`.

| Field | Required | Description |
|------|------|------|
| Display Name | ✓ | Agent name shown in the UI |
| Agent ID | ✓ | Target agent folder (lowercase letters/numbers/hyphens) |
| Chat ID | ✓ | Telegram ID of the peer (user or group) |
| Type | ✓ | `User` (DM) or `Group` (group chat) |
| Bot Token | Required for `User` | Dedicated bot token; optional for `Group` (uses default bot if blank) |

- **User type**: Bot Token is required, binding is written to `config.json`, and bot polling starts immediately.
- **Group type**: The bot must already be in the group; Bot Token is optional (falls back to default bot in `.env` if empty).

To delete, click the delete button on the agent card. `User` bindings are removed from `config.json` as well.

### CLI Configuration

```bash
# Add
senclaw channel telegram add \
  --token 123456:ABC-your-token \
  --user 987654321 \
  --group alice

# List
senclaw channel telegram list
senclaw channel list            # all channel types

# Remove (restart required)
senclaw channel telegram remove --token 123456:ABC-your-token
```

| Parameter | Required | Description |
|------|------|------|
| `--token` | ✓ | Bot token generated by BotFather |
| `--user` | ✓ | Admin Telegram User ID |
| `--group` | ✓ | Target agent folder |
| `--name` | | Optional display name |

> CLI add takes effect after restart; Web UI add is instant. Remove operations sync to `config.json` in both methods, so the bot will not be loaded on next startup.

> **Note**: `remove` only applies to Method 2. For the primary bot in Method 1 (`.env`), manually clear `TELEGRAM_BOT_TOKEN` and `ADMIN_TELEGRAM_USER_ID` in `.env`, then restart (falls back to web-only mode).

---

## Multiple Bots in One Folder

Two bots can bind to the same folder (same agent). In that case:

- Each bot polls Telegram independently.
- All messages are routed to the **same agent instance** and processed serially.
- Conversation history from both sides is **shared** (like a merged chat context).

Recommendation: unless you explicitly need a backup account, bind **one bot per folder** to avoid serial blocking and context pollution.
